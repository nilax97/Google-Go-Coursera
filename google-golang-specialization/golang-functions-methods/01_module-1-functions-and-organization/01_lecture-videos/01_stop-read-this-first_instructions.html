Before watching the lectures, please note the following:

  1. There is a small amount of controversy surrounding the expressions "pass by pointer" and "pass by reference." Just search the web for "Pass by Pointer vs. Pass by Reference in Go Programming" and you'll see for yourself! Be aware that our instructor uses "pass by reference" where some Go users would say "pass by pointer" or "pass a pointer."

  2. Video Lecture M1.1.4: At 4:07, the code appearing on the screen is missing square brackets in function foo as follows: foo(slide [] int).
